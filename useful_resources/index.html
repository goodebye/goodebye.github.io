TOOLS THAT I USE AND THINK ARE GOOD AND MOSTLY WELL DESIGNED AND MAY BE USEFUL FOR PEOPLE WHO AREN'T INCREDIBLY SAVVY YET BUT STILL HAVE SOME AMOUNT OF KNOW HOW AS FAR AS LIKE PROGRAMMING ET CETERA GOES

  • processing (free) (programming framework for interactive/visual art on The Personal Computer (primarily, u can technically take it online but i would recommend using p5.js for web things), used this for the art in gut flora and some installation stuff too)
  • p5.js (free) (programming framework for interactive/visual art on The Web, used this for my jenny holzer generator and the animations on my home page and other stuff too)
  • LÖVE (free) (programming framework for 2d games, using this for chroma solstice)
  • bitwig studio (free if u pirate it which isnt that hard) (digital audio software similar to ableton but available for linux and does a couple of other cool things...10/12 songs on "is it that bad" were made using bitwig and i use it p much exclusively now!!)
  • arduino (costs money for a kit but the software is free) (microcontroller that i use for my Electronics Projects including an instrument i am prototyping and a lightshow involving incandescent lamps for Live Performances, the platform is very easy to pick up, i can find u some cheap kits which can still do cool things if u email me !!!)
  • meteor (free) (web framework which i think is rly good for beginners to server-side dev to do cool things rly quickly, used it to make TXTUARY)
  • elementary os (free) (operating system which i hav been using for a long time)

BEGINNER'S RESOURCES FOR DIGITAL ART SH*t

  • the hour of code for processing by daniel shiffman - i tell everyone to do this that expresses even a passing interest in learning how to program. i talk more abt dan further down but he works so sososososo hard on making things that could be complicated accessible to many ppl w/ many diff backgrounds so if u can set aside maybe 45 minutes i recommend u do this ok !! you may want to also check out his book learning processing for further info (i can maybe help u find an ebook if u just email me or tweet at me or s/t)
  • how to make games with twine by auntie pixelante / anna anthropy - twine is a platform for making interactive fiction games for ppl w/ no programming experience !! it's all visual so u dont have to write a lick of code if u dont want to !! a lot of great stuff has been made w/ twine u should just google it tbh
  • flick - this is a lil more esoteric initially than the other things but the core concepts are actually super simple and u can make a goofy game in no time. click the help button and mess with the buttons until you get a feel for it. try out any of the games at the flick game gallery if you feel confused about the format.

"TECHNICAL" BOOKS WHICH ARE ACTUALLY GOOD AT RELAYING INFORMATION

  • the nature of code (pay what u want ebook) by daniel shiffman - this is not an intro programming text (if u want something for beginners you should check out learning processing as i mentioned above (as always email or tweet at me or s/t if u cant find an ebook dl somewhere) by the same person) but it's mayb one of the most best goodest Creative Coding texts for intermediate programmers of All Time....daniel shiffman takes such great care in his Pedagogy, the way he develops n explains systems n models is so thoughtful i love it ok.
  • programming in lua (html textbook) by roberto ierusalimschy (co-creator of lua) - do u want to know something shocking? this is the only programming resource i've ever encountered which defaults to she/her pronouns w/r/t the Hypothetical Programmer. this is not a beginner's programming book but it's a very careful, thoughtful, accessible survey of the only programming language i truly love, lua. i really love it...i actually have read this twice cover to cover. lua is porteguese for moon n it is worthy of the title imo
  • beginning node.js (amazon link) by basarat syed - i only just started reading this but i actually understood closures for the first time in the 1.5 yrs since they were initially explained to me after reading the chapters on the basics of javascript. rly good so far imo !! email me if u have trouble finding an ebook.......just email me.........
  • theory and technique of electronic music (pdf link) by miller puckette - this is easily the densest of the texts listed here which is why it's at the end. miller puckette created both max and pure data (pd) which are two of the world's most well known audio programming platforms !!!! this book is rly math heavy and i've barely made a dent in it but it rly digs into the atomic aspects of digital sound which rarely get talked abt and i bet if more ppl read it music would increase in quality by approximately 10-15%
